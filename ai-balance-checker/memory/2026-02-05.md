3. 风险因素：竞争加剧、平台封号风险、客户流失

## 微信 & 抖音客服接入指南

已创建详细的接入指南文档：`微信抖音客服接入指南.md`

### 文档内容概述

#### 微信客服接入
1. 账号类型对比：订阅号（个人/自媒体）、服务号（企业）、企业微信
2. 客服接口开通流程
3. 技术实现方案：
   - 方案 A：服务器推送（推荐）
   - 方案 B：无服务器方案（SaaS）
4. 关键技术注意事项：Token 验证、消息加密、消息去重、主动推送限制

#### 抖音客服接入
1. 账号类型对比：个人号、企业号、蓝 V 认证
2. 客服接口开通：企业号（推荐）、蓝 V 认证（更高信任度）
3. 费用结构：基础 API 使用费 + 高级功能订阅
4. 技术实现方案：机器人 API（推荐）、第三方客服系统
5. 关键技术注意事项：内容审核、消息发送限制、机器人权限

#### 对接技术选型
基于 Dify 的推荐架构：
- 前端：Next.js / Vue 3（管理后台）
- 后端：Dify（开源部署）+ 定制 API
- LLM：千问 API（便宜、中文友好）
- 数据库：PostgreSQL + Redis
- 存储：阿里云 OSS / MinIO

#### 成本优化策略
- LLM API：使用千问 API（¥0.008/1k tokens），使用缓存减少重复调用
- 服务器：轻量级云服务器（阿里云 2C4G，¥300/月）
- 存储：按需使用 OSS

#### 接入流程时间线
- 微信接入：4 周（账号申请 → 域名备案 → 部署测试）
- 抖音接入：4 周（企业号申请 → 接口权限 → 测试上线）

#### 风险提示
- 平台风险：微信封号、抖音内容审核严格
- 合规建议：提供客服标识、不发送营销骚扰、建立人工客服入口

#### 快速检查清单
接入前必须确认：
- [x] 已申请企业微信账号（或认证公众号）
- [x] 已申请抖音企业号（个人号无法开通客服接口）
- [x] 有公网 IP 服务器或云服务器
- [x] 服务器已配置 HTTPS（微信必须）
- [x] 已准备好 LLM API 密钥
- [x] 后端语言栈确认（推荐 Python + FastAPI）
- [x] 数据库类型确认（推荐 PostgreSQL）
- [x] WebSocket 库确认（Socket.IO / websockets）
